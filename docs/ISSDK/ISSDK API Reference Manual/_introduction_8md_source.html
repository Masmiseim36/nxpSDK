1 Introduction {#index}
2 =========================
3 
4 <P>ISSDK 1.6 is designed as a middleware component inside the Kinetis SDK 2.2 ecosystem.
5 This means that ISSDK 1.6 takes advantage of the deployment infrastructure, device specific drivers,
6 and project creation tools to supply customers with a consistent “out-of-box” experience comparable
7 to any other SDK 2.2 software. In addition to the base enablement, the ISSDK is augmented with demo
8 sensor and algorithm example projects for supported board-kits, and API documentation to help users
9 quickly leverage the support of the ISSDK 1.6.
10 
11 <P>ISSDK 1.6 is fully integrated into the MCUXpresso Web & SDK Builder (Previously known as
12 Kinetis Expert (KEx)) delivery system. MCUXpresso Web & SDK Builder is a cloud-based system used to
13 build Kinetis SDK 2.2 packages. MCUXpresso Web & SDK Builder includes both cloud and locally based
14 tools to collect and build projects from the Kinetis SDK repositories.
15 KSDK 2.2 is built using a hierarchy of deployed Git repositories. Specific project codebases are built
16 through the online tool. A given codebase is specified by its target (i.e., the device, board, or kit desired),
17 the version of KSDK 2.x, the supported IDEs (i.e., MCUXpresso IDE (RedEye), KDS, IAR, Keil, etc.) and the
18 target Host OS(i.e., Windows, Mac, or Linux).
19 There is also an option to build projects locally using a pre-installed KSDK and the Project Builder feature of KDS.
20 
21 <P> For each Sensor Driver and communications channel there is a set of Driver Examples which demonstrate the
22 usage of the Sensor Driver over a variety of conditions that make sense for the sensor.
