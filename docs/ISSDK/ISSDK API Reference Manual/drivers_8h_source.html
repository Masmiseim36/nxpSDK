45 /// @name SysTick Macros
46 /// The ARM SysTick counter is used to time various fusion options. Timings
47 /// are then conveyed to the NXP Sensor Fusion Toolbox, where they are displayed
48 /// for the developer. These functions should be portable to any ARM M0+, M3,
49 /// M4 or M4F device. If you are using a different CPU architecture, you will
50 /// need to provide an equivalent set of macros, remove the macro calls from
51 /// the fusion routines, or define a set of empty macros.
52 ///@{

59 /// @name Sensor Drivers
60 /// Each physical sensor must be provided with one initialization function
61 /// and one "read" function. These must be installed by the user using the
62 /// installSensor method defined in SensorFusionGlobals. By "physical sensor",
63 /// we mean either individual sensor type (such as a 3-axis accelerometer) or
64 /// a combo-sensor such as the NXP FXOS8700 6-axis accel plus mag. The init()
65 /// function for each sensor is responsible for initializing all sensors contained
66 /// in that package. The read() function is responsible for reading those same
67 /// sensors and moving the results into the standard structures contained within
68 /// the SensorFusionGlobals object.
