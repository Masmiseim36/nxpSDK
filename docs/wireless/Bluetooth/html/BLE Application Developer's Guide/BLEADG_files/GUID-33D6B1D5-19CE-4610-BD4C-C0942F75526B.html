Deep sleep mode 4

This is the lowest power mode of all. It was designed to be used when the BLE stack is idle. In this mode, the MCU enters VLLS0/VLLS1 and BLE Link Layer remains idle. The SoC wakes up from this mode by GPIOs configured as wake-up source in BOARD_LLWU_PIN_ENABLE_BITMAP, using LLWU module. No synchronization for low-power timers is made since this deep sleep mode is exited through reset sequence. There are two defines that configures this mode:

  • cPWR_DCDC_InBypass configures the VLLS mode used. If this define is TRUE the MCU enters VLLS0, otherwise MCU enters VLLS1 since VLLS0 is not allowed in DCDC buck or boost mode.
  • cPWR_POR_DisabledInVLLS0. This define only has meaning if cPWR_DCDC_InBypass is TRUE so the MCU enters VLLS0 mode. If TRUE, this define disables POR circuit in VLLS0 making this deep sleep mode lowest power mode possible.
