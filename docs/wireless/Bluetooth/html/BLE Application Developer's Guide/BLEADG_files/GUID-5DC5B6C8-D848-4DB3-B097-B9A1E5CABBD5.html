GAP connection event

The GAP Connection Event is triggered by the Host Stack and sent to the via the connection callback. Before any application-specific interactions, the Connection Manager callback is called to handle common application events, such as device connect, disconnect or pairing related requests It is called inside the registered connection like below:

staticvoidBleApp_ConnectionCallback ( deviceId_t peerDeviceId, gapConnectionEvent_t * pConnectionEvent)
{
    /* Connection Manager to handle Host Stack interactions */
    BleConnManager_GapPeripheralEvent(peerDeviceId, pConnectionEvent);

    switch (pConnectionEvent-> eventType )
    { 
        ... 
    }
}

It is strongly recommended that the application developer use the app.c module to add custom code.
