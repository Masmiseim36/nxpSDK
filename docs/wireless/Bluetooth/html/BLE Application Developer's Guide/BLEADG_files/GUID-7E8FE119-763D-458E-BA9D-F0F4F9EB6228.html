Introduction

The Bluetooth 4.2 Host Stack introduces support for the Enhanced Privacy feature.

Privacy can be enabled either in the Host or in the Controller:

  • Host Privacy consists of:
    • Periodically regenerating a random address (Resolvable or Non-Resolvable Private Address) inside the Host and the applying it into the Controller
    • Keeping a list of peer IRKs in the Host and trying to resolve any incoming RPA
  • Controller Privacy, introduced by Bluetooth 4.2, consists of writing the local IRK in the Controller, together with all known peer IRKs, and letting the Controller perform hardware, fully automatic IRK generation and resolution

Either Host Privacy or Controller Privacy can be enabled at any time. Trying to enable one while the other is in progress generates a gBleInvalidState_c error. The same error is returned when trying to enable the same privacy type twice, or when trying to disable privacy when it is not enabled.
