Deep sleep modes for Kinetis MCU-based platforms

See the Kinetis Wireless Dual Mode (Bluetooth LE and IEEE® 802.15.4) Microcontrollers Low-power Library Overview section of Connectivity Framework Reference Manual for the list of supported deep sleep modes and their description.

A summary of the available power modes can be found in the table below:

Table 1. Available power modes
Low-Power Mode Required State Wake Up Sources
MCU Bluetooth LE Link Layer GPIO Bluetooth LE LL LPTMR DCDC** UART
1 LLS3 DSM x x x    
2 LLS2 IDLE x   x x  
3 LLS3 IDLE x   x x  
4 VLLS0/1* IDLE x     x  
5 VLLS2 IDLE x     x  
6 STOP IDLE/RUN x x x x x

* VLLS0 if DCDC bypassed/ VLLS1 otherwise

** Available in buck mode only

The application can decide to configure cMCU_SleepDuringBleEvents to allow better power consumption when the LL performs an action. For instance, if the user configures the system to use low power mode 1 and this configuration is enabled, during Bluetooth Low Energy events, the low power mode 6 will be used. After LL completed the event, the user selected low power mode will be restored, as in this example low power mode 1.
