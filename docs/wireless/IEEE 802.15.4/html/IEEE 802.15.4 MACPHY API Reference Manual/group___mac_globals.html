          #define gMacCslTableSize_c
        

      





The maximum number of entries in the CSL Synchronization table that specifies how many devices can the MAC layer store synchronization details for (CSL listening period and timestamp of next channel sample). 











      

        

          #define gMacCslMaxSequencesCnt_c
        

      





This constant defines the maximum number of CSL sequences that can be present in the MAC layer when CSL is enabled. 











      

        

          #define gMacRitTableSize_c
        

      





This constant defines the maximum number of entries in the RIT table that specifies how many devices can the MAC layer store listening schedule for (timestamp of last received RIT Data Request command and listening schedule). 











      

        

          #define gMacRitMaxSequencesCnt_c
        

      





This constant defines the maximum number of RIT sequences that can be present in the MAC layer when RIT is enabled. 











      

        

          #define gMacHoppingSequenceTableEntries_c
        

      





This constant defines the maximum number of channels that the MAC layer can store to hop on when TSCH is enabled. 











      

        

          #define gMacSlotframeTableEntries_c
        

      





This constant defines the maximum number of TSCH slotframes that the MAC layer can store. 











      

        

          #define gMacLinkTableEntries_c
        

      





This constant defines the maximum number of TSCH links that the MAC layer can store. 











      

        

          #define gMacTschNeighborTableEntries_c
        

      





This constant defines the maximum number of TSCH time source neighbors that the MAC layer can store when TSCH is enabled. 


These are configured through the MLME-KEEP-ALIVE.Request primitive, and they must have the Rx Link with timekeeping option set for the same node address. 











      

        

          #define gMacTschMaxPanCoordSync_c
        

      





This constant defines the maximum number of PAN Coordinators that the MAC layer can store during the TSCH Passive Scan procedure. 


This information is required for TSCH service enablement following the scan procedure. 











      

        

          #define gMacTschMaxTransactions_c
        

      





This constant defines the maximum number of Data Tx sequences that can be present in the MAC layer when TSCH is enabled.
