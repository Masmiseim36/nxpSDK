Enumeration Type Documentation






      

        

          enum phyStatus_t
        

      





This type enumerates all the possible statuses of primitives that require passing a status to the MAC Layer. 




Enumerator
gPhyChannelBusy_c 

The CCA attempt has detected a busy channel. 




gPhyBusyRx_c 

The transceiver is asked to change its state while receiving. 




gPhyBusyTx_c 

The transceiver is asked to change its state while transmitting. 




gPhyChannelIdle_c 

The CCA attempt has detected an idle channel. 




gPhyInvalidParameter_c 

A SET request was issued with a parameter in the primitive that is out of the valid range. 




gPhyRxOn_c 

The transceiver is in the receiver enabled state. 




gPhySuccess_c 

A SET/GET, an ED operation, a data request, an indirect queue insert, or a transceiver state change was successful. 




gPhyTRxOff_c 

The transceiver is in the transceiver disabled state. 




gPhyTxOn_c 

The transceiver is in the transmitter enabled state. 




gPhyUnsupportedAttribute_c 

A SET/GET request was issued with the identifier of an attribute that is not supported. 




gPhyReadOnly_c 

A SET request was issued with the identifier of an attribute that is read-only. 




gPhyIndexUsed_c 

The indirect queue insert operation has detected an used index. 




gPhyNoAck_c 

No ACK was received for the last transmission. 




gPhyFramePending_c 

The ACK of a Data Request frame indicates a pending frame in the coordinator’s indirect TX queue. 




gPhyBusy_c 

The current request cannot be handled, because the PHY is busy. 




gPhyInvalidPrimitive_c 

The set was not completed, because the primitive is not in the valid range.

Enumerator

          enum phyAckRequired_t
        

      





This type is used to filter the next received frames, and to accept only Ack frames. 




Enumerator
gPhyRxAckRqd_c 

A receive Ack frame is expected to follow the transmit frame (non-Ack frames are rejected) 




gPhyNoAckRqd_c 

An ordinary receive frame (any type of frame) follows the transmit frame. 















      

        

          enum phyPibId_t
        

      





This type enumerates all PHY PIB IDs. 


The PIBs can be read and written to by the upper layer, and are used to configure certain parameters and modes of operation for the PHY Layer. The PIBs that refer to physical parameters, such as carrier frequency or transmission power, are usually mirrored in hardware, but in case the IEEE 802.15.4 hardware acceleration is used, there are protocol-oriented PIBs also stored in hardware, such as addresses, PAN ID, promiscuous mode setting, and so on. 




Enumerator
gPhyPibCurrentChannel_c 

The channel currently used. 




gPhyPibCurrentPage_c 

The channel page currently used. 




gPhyPibTransmitPower_c 

The power used for TX operations. 




gPhyPibLongAddress_c 

The MAC long address to be used by the PHY’s source address matching feature. 




gPhyPibShortAddress_c 

The MAC short address to be used by the PHY’s source address matching feature. 




gPhyPibPanId_c 

The MAC PAN ID to be used by the PHY’s source address matching feature. 




gPhyPibPanCoordinator_c 

Indicates if the device is a PAN coordinator or not. 




gPhyPibSrcAddrEnable_c 

Enables or disables the PHY’s source address matching feature. 




gPhyPibPromiscuousMode_c 

Selects between normal, promiscuous and active promiscuous mode. 




gPhyPibAutoAckEnable_c 

Enables or disables automatic transmission of ACK frames. 




gPhyPibFrameVersion_c 

Used in checking for allowed frame versions (0x00 - any version accepted, 0x01 - accept Frame Version 0 packets (2003 compliant), 0x02 - accept Frame Version 1 packets (2006 compliant), 0x03 - accept Frame Version 0 and 1 packets). 




gPhyPibFrameEnable_c 

Used for enabling or disabling reception of MAC frames. 




gPhyPibAckFramePending_c 

Used to copy its contents to the outgoing ACK frame's Frame Pending field as a response to a received Data Request frame with Source Address Matching disabled. 




gPhyPibRxOnWhenIdle 

Enable RX when the radio is IDLE. 




gPhyPibFrameWaitTime_c 

The number of symbols the RX should be on after receiving an ACK with FP=1. 




gPhyPibDeferTxIfRxBusy_c 

If a packet is being received, transmission is deferred. 




gPhyPibLastTxAckFP_c 

Indicates, whether the last ACK sent to the MAC Data Request had the frame pending flag set.
